# Contributing to Math Flash Cards 🧮

Thank you for your interest in contributing to Math Flash Cards! This project is designed to help learners practice math in a fun, engaging way. We welcome contributions from developers of all skill levels.

## 🤝 How to Contribute

### Reporting Bugs
- Use the GitHub issue tracker
- Include steps to reproduce the bug
- Specify your browser and operating system
- Add screenshots if applicable

### Suggesting Features
- Open a discussion or issue
- Describe the feature and its benefits
- Consider the educational value and user experience

### Code Contributions
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## 🎯 Areas for Contribution

### Frontend Development
- **UI/UX Improvements**: Better visual design, animations, accessibility
- **New Game Modes**: Different difficulty levels, new operation types
- **Mobile Optimization**: Enhanced mobile experience
- **Theme System**: Dark mode, color schemes, customization options

### Educational Features
- **Learning Analytics**: Better progress tracking, learning insights
- **Adaptive Difficulty**: Questions that adjust to user skill level
- **Multi-language Support**: Internationalization for global users
- **Accessibility**: Screen reader support, keyboard navigation

### Technical Improvements
- **Performance**: Code optimization, faster loading
- **Progressive Web App**: Offline support, app-like experience
- **Testing**: Unit tests, integration tests, browser testing
- **Documentation**: Code comments, API documentation

## 🛠️ Development Guidelines

### Code Style
- Use consistent indentation (2 spaces)
- Follow JavaScript ES6+ best practices
- Write clear, self-documenting code
- Add comments for complex logic

### Testing
- Test in multiple browsers (Chrome, Firefox, Safari, Edge)
- Verify mobile responsiveness
- Test accessibility features
- Ensure no console errors

### Commit Messages
- Use clear, descriptive commit messages
- Reference issues when applicable
- Follow conventional commit format when possible

## 📋 Pull Request Process

1. **Fork and Clone**: Fork the repository and clone your fork
2. **Create Branch**: Create a descriptive branch name
3. **Make Changes**: Implement your feature or fix
4. **Test Thoroughly**: Ensure everything works correctly
5. **Update Documentation**: Update README or other docs if needed
6. **Submit PR**: Create a pull request with a clear description

### PR Description Template
```markdown
## Description
Brief description of changes

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Documentation update
- [ ] Performance improvement
- [ ] Accessibility improvement

## Testing
- [ ] Tested in Chrome
- [ ] Tested in Firefox
- [ ] Tested in Safari
- [ ] Tested on mobile
- [ ] No console errors

## Screenshots (if applicable)
Add screenshots to help explain your changes
```

## 🎨 Design Principles

### Educational Focus
- Keep the learning experience engaging and positive
- Provide clear, encouraging feedback
- Make progress visible and motivating
- Ensure accessibility for all learners

### Technical Simplicity
- Maintain the "no dependencies" philosophy
- Keep the codebase lightweight and fast
- Use modern web standards
- Prioritize browser compatibility

### User Experience
- Intuitive interface design
- Responsive across all devices
- Fast loading and smooth interactions
- Clear visual hierarchy

## 🐛 Bug Reports

When reporting bugs, please include:

1. **Browser and Version**: Which browser you're using
2. **Operating System**: Windows, macOS, Linux, mobile
3. **Steps to Reproduce**: Clear steps to recreate the issue
4. **Expected Behavior**: What should happen
5. **Actual Behavior**: What actually happens
6. **Screenshots**: If applicable, include screenshots
7. **Console Errors**: Any error messages in the browser console

## 💡 Feature Requests

When suggesting features, consider:

- **Educational Value**: How does this help learners?
- **User Experience**: Is it intuitive and engaging?
- **Technical Feasibility**: Can it be implemented with current tech stack?
- **Accessibility**: Does it work for all users?
- **Performance**: Does it maintain fast loading times?

## 📞 Getting Help

- **Discussions**: Use GitHub Discussions for questions
- **Issues**: Open an issue for bugs or feature requests
- **Code Review**: Ask for help in pull requests

Contributors will be recognized in:
